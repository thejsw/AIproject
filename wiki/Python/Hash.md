# 📚 Today I Learned - Hash (해시)

## 🎯 Hash 접근법

다음과 같은 상황이라면 **Hash(`dict`, `set`)**를 먼저 떠올린다.

- 개수 세기 → `dict`
- 중복 제거 → `set`
- 존재 여부 확인 → `set` 또는 `dict`

### ✅ 문제 접근 순서

1. 제한사항을 보고 `O(N²)`이 가능한지 확인
2. 빠른 조회가 필요하면 Hash 사용 고려
3. 문제 유형에 맞는 자료구조 선택

---

## 🏃 완주하지 못한 선수

### 💡 핵심 아이디어

참가자 수를 저장한 뒤, 완주자를 차감한다.

### 📝 풀이 순서

1. `dict` 생성
2. 참가자 이름 개수 `+1`
3. 완주자 이름 개수 `-1`
4. 값이 `0`이 아닌 이름 반환

**Pattern:** 📊 Frequency Count (`dict`)

---

## 🎮 폰켓몬

### 💡 핵심 아이디어

중복을 제거한 뒤, 선택 가능한 마리 수와 비교한다.

### 📝 풀이 순서

1. 선택 가능한 수(`N // 2`) 계산
2. `set`으로 중복 제거
3. 종류 수 계산
4. `min(선택 가능 수, 종류 수)` 반환

**Pattern:** 🗂️ Deduplication (`set`)

---

## ✅ 해시 문제 체크리스트

- 개수를 세야 하는가? → `dict`
- 중복을 제거해야 하는가? → `set`
- 빠른 존재 여부 확인이 필요한가? → `set` / `dict`
- `O(N²)`이 불가능한가? → Hash 사용 고려
